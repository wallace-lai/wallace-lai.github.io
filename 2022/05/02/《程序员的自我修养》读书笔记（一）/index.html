一、目标文件格式

ELF文件的大致结构如下

1.1 ELF文件头

使用readelf命令读取simple.o的elf文件头信息如下。

64位ELF Header的结构定义在/usr/include/elf.h中，各字段的含义如下。

  • e_ident：包含了文件头中的Magic，Class，Data，Version，OS/ABI和ABI Version的内容。这16个字节的内容解析如下。

  • e_type：ELF文件类型。常见的值有可重定位文件ET_REL、可执行文件ET_EXEC、共享目标文件ET_DYN和CoreDump文件ET_CORE。
  • e_machine：对应的机器类型。比如常见的EM_386、EM_860等。
  • e_version：ELF版本号，一般为常数1
  • e_entry：规定ELF程序的入口虚拟地址，操作系统在加载完该程序后从这个地址开始执行进程的指令。可重定位文件一般没有这个入口地址，值为0
  • e_phoff：TODO
  • e_shoff：段表在文件中的偏移，simple.o中该值为1088表明段表是从该文件中的第1089个字节开始的
  • e_flags：ELF标志位，用来标识一些ELF文件平台相关的属性
  • e_ehsize：ELF文件头的大小，simple.o中该值为64B大小
  • e_phentsize：TODO
  • e_phnum：段表描述符的数量，simple.o中该值为13，表示该文件中含有13个段
  • e_shstrndx：段表中字符串表在段表中的下标

1.3 ELF中各段

使用objdump -h可以把ELF文件中各个段的基本信息给打印出来，使用size则可以将ELF文件中代码段、数据段和bss段的大小给打印出来。simple.o的各段信息如下所示。

.text段

.text段存储的是代码对应的二进制指令。使用objdump -ds可以将.text段的内容给反汇编出来，结果如下。可以看到.text段中的内容是func1和main这两个函数的二进制代码。

.data段

.data段存放的是初始化了的全局静态变量和局部静态变量，它们对应了代码中的g_initVar和staticVar1。总共8个字节大小。

.bss段

.bss段存放的是未初始化的全局变量和局部静态变量，注意到simple.o中的.bss段只有4B的大小。其实它对应的是staticVar2。全局变量g_uninitVar则是一个未定义的COMMON符号，它在simple.o被链接成可执行程序时在.bss段中分配空间。

其他常见的段

段名 描述
.rodata1 存放只读数据，和.rodata一样
.comment 存放编译器版本信息，比如”.GCC: (Ubuntu 9.4.0-1ubuntu1~18.04) 9.4.0.”
.debug 存调试信息
.dynamic 动态链接信息
.hash 符号哈希表
.line 调试时的行号表
.note 额外的编译器信息，比如程序的公司名、发布版本号等
.strtab 字符串表，用于存储ELF文件中用到的各种字符串
.symtab 段名表
.plt / .got 动态链接的跳转表和全局入口表
.init / .fini 程序的初始化和终结代码段

常用命令

（1）file

  • 查看目标文件的类型（可重定位文件、可执行文件、共享目标文件、核心转储文件）

（1）size

  • 查看目标文件中data、text和bss段的大小

（1）objdump

  • -r：读取目标文件的重定位表
  • -h：查看目标文件各个段的基本信息（段表信息）
  • -d：对目标文件的text段进行反汇编

如果要将目标文件中所有段的内容都进行反汇编，请加上-s参数；如果要得到更加详细的信息，请加上-x参数。

  • -t：查看目标文件的符号表，类似nm或者readelf -s命令的功能

（2）readelf

  • -s：查看目标文件的符号表，类似objdump -t或者nm命令的功能
  • -h：查看目标文件的ELF头信息
  • -S：大写的S表示查看目标文件中的段表信息（注意重定位表、段表、符号表这三者之间的区别）

（3）nm

nm命令专门用于查看目标文件中的符号表（可以用objdump -t或者readelf -s命令代替）。常用的选项如下：

  • -C：反名字重整，即显示原始的函数名而不是经过名字重整后的函数名
  • -l：打印符号所在的源文件和源文件中的行号

可以看到a.o中有两个符号shared和swap是UNDEFINED的。

（3）ar

  • -t：查看.a文件中包含了哪些.o
  • -x：解压.a文件
